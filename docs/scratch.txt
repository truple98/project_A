
[프로젝트: 텍스트 기반 RPG/TCG 게임 with LLM 지원 시스템]

📅 작성일: 2025-07-13


────────────────────────────────────────────
📌 1. 개발 기술 스택 구성
────────────────────────────────────────────

● 프론트엔드
  - 모바일 앱: React Native (TypeScript)
  - 웹: React (TypeScript)

● 백엔드
  - Node.js (TypeScript)
  - Express 또는 Fastify 기반 API 서버
  - LLM 호출용 API 라우팅 레이어 포함

● 배포 대상 플랫폼
  - 모바일 (iOS/Android): React Native 앱 패키징
  - 웹: React 앱 배포 (Vercel 등)
  - PC (스팀 배포): Electron 또는 Tauri로 React 앱 래핑

────────────────────────────────────────────
📌 2. 로컬 LLM 전략
────────────────────────────────────────────

● 사용 목적
  - 게임 내 리액션, 묘사, 선택지 설명, NPC 대사 등 생성
  - 반복 플레이 시 새로운 경험 제공

● 추천 모델
  - ✅ Mistral 7B Instruct → 경량, 반응 속도 빠름, 퀄리티 적절
  - ✅ OpenChat 3.5 → 대화형 묘사에 적합, CPU 가능
  - ✅ MythoMax 13B → 감성 서사에 강함, GPU 필요

● 실행 도구
  - Ollama, llama.cpp, text-generation-webui, vLLM 등
  - HTTP API 형태로 게임 백엔드에서 연결 가능

● 클라우드 서버 운영 시 고려사항
  - AWS EC2 (g4dn/g5 시리즈) 또는 GPU 인스턴스
  - 캐시 시스템과 사전 생성 데이터 연계 필요
  - 비용 절감을 위해 GPT-4 API 병행 사용 가능

────────────────────────────────────────────
📌 3. LLM 사용 전략 설계
────────────────────────────────────────────

● 프롬프트 목적별 분리 권장 (LLMRouter 설계)

  - 목적별:
    - reaction: 리액션 묘사
    - narration: 상황 설명
    - npc: NPC 반응
    - choice: 선택지 묘사

● 비용 고려 사항 (GPT-4o 기준)
  - 호출당 약 0.0025 USD (0.33원)
  - 유저 1인당 1일 약 10회 호출 → 0.0075 USD
  - DAU 1,000명 → 월 $225 예상
  - 캐싱, fallback (GPT-3.5), 프리미엄 분리 운영 전략 필요

────────────────────────────────────────────
📌 4. 콘텐츠 생성 설계
────────────────────────────────────────────

● 노드 기반 진행 구조 + LLM 생성 지원

  - 스토리 분기/선택지/랜덤 인카운터 구조는 기존대로 설계
  - 단, 일부 묘사나 보상/리액션 등은 LLM이 결정

● 예시:
  - 플레이어 성향 (차분한 검사 vs 호전적인 마법사)에 따라 리액션 묘사 차별
  - 성향과 선택지 일치 시 시너지 보너스 제공
  - 엔딩은 고정되어도 과정은 LLM으로 매회 변주 가능

📌 5. 향후 확장 고려사항
────────────────────────────────────────────
